- [ ] "abc123" 입력 → 노란 바 2칸, "약함", "영문자 + 숫자" 체크 (실패)
  - `약함` 표시는 확인했지만 실제 칸 수까지는 이 세션에서 검증하지 못했습니다.
- [x] "abc12345" 입력 → 주황 바 3칸, "보통", "8자 이상 + 영문자 + 숫자" 체크
- [x] "abc1234!" 입력 → 초록 바 4칸, "강함", 4개 조건 모두 체크

### 2-6. 비밀번호 유효성 검사

### 2-8. 회원가입 성공 (백엔드 연동)
- [x] 유효한 정보 입력 후 "회원가입" 클릭
- [x] 버튼이 로딩 상태(스피너)로 변경되는지 확인
- [x] Network 탭: `POST /api/v1/auth/register` 요청 → 201 응답
- [x] Network 탭: 자동으로 `POST /api/v1/auth/login` 요청 → 200 응답 (자동 로그인)
- [ ] Network 탭: `GET /api/v1/users/me` 요청 → 200 응답 (프로필 조회) (실패)

- [ ] 토스트: "회원가입이 완료되었습니다." 표시 (실패)
  - 성공 토스트를 확인하지 못했습니다. `users/me` 401 이후 `/login`으로 되돌아갑니다.
- [ ] `/`(홈, 메인 화면)으로 리다이렉트 (실패)
  - 자동 로그인은 호출되지만 최종적으로 `/login`으로 돌아가므로 완료 플로우가 끊깁니다.
- [ ] Application 탭: localStorage/sessionStorage에 토큰이 저장되지 않았는지 확인 (메모리 전용) (미실행)

## 4. OAuth 소셜 로그인

### 4-1. Google 로그인 버튼 동작
- [x] 로그인 페이지에서 "Google로 로그인" 클릭
- [x] Network 탭: `GET /api/v1/auth/oauth/google?redirectUri=http://localhost:3000/oauth/callback` 요청
- [x] 백엔드 응답에 `authorizationUrl`과 `state` 포함 확인
- [x] Google 로그인 페이지(accounts.google.com)로 리다이렉트
  - Google 인증 URL에 `client_id=`가 비어 있어 `Missing required parameter: client_id` 400 오류 페이지가 열립니다.

### 4-2. Kakao 로그인 버튼 동작
- [x] 로그인 페이지에서 "Kakao로 로그인" 클릭

- [x] Kakao 로그인 페이지(kauth.kakao.com)로 리다이렉트
  - Kakao authorize URL의 `client_id=`도 비어 있습니다.

### 4-3. OAuth 콜백 처리 (/oauth/callback)
- [ ] 소셜 로그인 완료 후 `/oauth/callback?code=...&state=...&provider=google` 형태로 돌아오는지 확인 (미실행)
- [ ] 콜백 페이지에서 "로그인 처리 중..." 스피너 표시 (미실행)
- [ ] Network 탭: `GET /api/v1/auth/oauth/google/callback?code=...&state=...&redirectUri=...` 요청 (미실행)
- [ ] Network 탭: `GET /api/v1/users/me` 요청 (미실행)
- [ ] `/`(홈)으로 리다이렉트 (미실행)
- [ ] 이후 새로고침 시 비로그인 상태로 돌아감 (토큰이 메모리 전용이므로 정상 동작) (미실행)

### 4-4. OAuth 콜백 실패
- [x] `/oauth/callback` 에 파라미터 없이 직접 접근 → 토스트: "올바르지 않은 인증 요청입니다." + `/login` 리다이렉트
- [x] `/oauth/callback?code=invalid&state=invalid&provider=google` → 토스트: "소셜 로그인에 실패했습니다." + `/login` 리다이렉트

### 4-5. 회원가입 페이지 소셜 버튼
