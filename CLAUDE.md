# Tag-Me Frontend 개발 원칙

> 태그 기반 소셜 일기 플랫폼 — Next.js 프론트엔드
> UI의 경우 tagdiary-minimal.jsx 참고

---

## 기술 스택

- **Framework**: Next.js (App Router, `src/app/`)
- **Language**: TypeScript (strict mode)
- **Styling**: TailwindCSS
- **마인드맵 시각화**: D3.js
- **에러 트래킹**: Sentry

---

## 렌더링 전략 (반드시 준수)

| 전략 | 적용 대상 | 이유 |
|------|-----------|------|
| **SSR** | 공개 피드, 사용자 프로필 | SEO 및 초기 로딩 속도 |
| **CSR** | 개인 일기장, 마인드맵 | 인증 필요 + 인터랙티브 시각화 |
| **ISR** | 트렌딩 태그 페이지 | 1시간 단위 갱신, 서버 부하 최소화 |
| **API Routes** | 이미지 리사이징 프록시 | S3 Pre-signed URL 발급 중계 |

---

## API 통신 규칙

- 백엔드 API는 RESTful, URI 패턴: `/api/v1/{resource}` (명사, 복수형)
- 응답 형식(Envelope): `{ success, data, error, timestamp }`
- 에러 코드는 도메인별 커스텀 코드 사용 (예: `DIARY_001`, `TAG_002`)
- 페이지네이션: 피드/검색은 **Cursor 기반**, 관리용은 Offset 기반
- 인증: Access Token (15분) + Refresh Token (7일, HttpOnly Cookie)
- Access Token 만료 시 자동 갱신 로직 필수

## 도메인 경계 (Bounded Context)

프론트엔드 코드도 백엔드의 도메인 경계를 따른다. 각 도메인의 관심사를 분리하여 구성한다.

| 도메인 | 담당 영역 |
|--------|-----------|
| **Identity** | 회원가입, 로그인, OAuth 소셜 로그인 (Google, Kakao) |
| **Diary** | 개인 일기 CRUD, 태그 입력, 스트릭, 오늘의 태그 프롬프트, 회고 카드 |
| **Feed** | 공개 게시글, 좋아요, 댓글, 일기 완성 후 피드 추천, 익명 공감 피드 |
| **Tag** | 태그 자동완성, 태그 메타데이터, 써클, 챌린지 |
| **Mindmap** | D3.js 태그 시각화, 기간/출처 필터 |
| **Social** | 팔로우, 유사 사용자 추천, 1:1 태그 친구 매칭 |
| **Search** | 통합 검색 (제목+본문+태그), 검색어 자동완성, 태그 하이라이팅 |
| **Notification** | 스트릭 알림, 회고 카드, 트렌딩 알림 |

---

## 태그 시스템 핵심 규칙

- 태그는 직접 작성 태그와 인터랙션(좋아요/댓글) 태그를 **시각적으로 구분**한다
  - 직접 작성: 진한 색
  - 좋아요/댓글 수집: 연한 색 + 아이콘 구분
- 마인드맵 노드 색상은 `source` (diary / post / like / comment)로 결정
- 마인드맵 엣지 스타일은 `source_type`으로 결정
- 일기를 쓰지 않아도 좋아요/댓글만으로 마인드맵이 형성되어야 함 (진입 부담 최소화)

---

## 이미지 처리

- `next/image` 사용 필수 (WebP 자동 변환, lazy loading)
- `next/font`로 폰트 사전 로드
- 이미지 업로드: S3 Pre-signed URL 방식 (서버 경유 금지, 클라이언트 직접 업로드)
- 단일 이미지 최대 10MB, 게시글당 최대 10장

---

## 보안 규칙

- OAuth 2.0 소셜 로그인 시 CSRF state 파라미터 필수
- 타인 일기 URL 직접 접근 차단 (본인 소유 확인)
- 익명 공감 피드는 집계 수치만 노출, 개별 사용자 식별 불가
- 민감 데이터(이메일 등)는 프론트에서 마스킹 처리
- credentials, API 키 등 코드 하드코딩 금지

## 리텐션 기능 (UX 설계 시 반드시 고려)

| 기능 | 핵심 UX |
|------|---------|
| 스트릭 & 습관 트래커 | 연속 기록 일수 시각화, 끊기면 복귀 유도 |
| 오늘의 태그 프롬프트 | 매일 태그 1개 제안 → 기록 시작 장벽 제거 |
| 회고 카드 "1년 전 오늘" | 과거 같은 날 기록을 카드로 노출 |
| 일기 완성 후 피드 추천 | 저장 완료 화면 하단 "이런 이야기도 있어요" 섹션 |
| 익명 공감 피드 | "오늘 #번아웃을 기록한 사람이 47명" 형태 |
| 월간 리포트 카드 | TOP 태그/활동 통계 인포그래픽, SNS 공유 기능 |

---

## 검색 UX

- 통합 검색: 일기(Private, 본인만) + 피드(Public) 동시 검색
- 검색 결과 관련도순 정렬 (제목 ×3, 태그 ×2, 본문 ×1 가중치)
- 일치 키워드 하이라이팅 필수
- 검색창 입력 중 태그/제목 자동완성 제안
- 필터: 기간, 태그, 작성자 조합

## 코드 컨벤션

- 컴포넌트: PascalCase (`DiaryCard.tsx`)
- 유틸/훅: camelCase (`useDiary.ts`, `formatDate.ts`)
- 타입 정의: 도메인별로 분리 관리
- API 호출 함수: 도메인별 모듈로 분리 (`api/diary.ts`, `api/feed.ts`)
